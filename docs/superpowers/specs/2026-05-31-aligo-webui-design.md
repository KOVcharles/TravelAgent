# Hommey 商旅助手 - Web UI 设计文档

## 概述

为 Hommey 商旅助手重新设计 Web 界面，替代现有的 Gradio 界面。采用深色模式 SaaS 风格，提供商务、专业、高效的用户体验。

## 技术栈

| 层 | 技术 | 选型理由 |
|---|------|---------|
| 后端框架 | FastAPI | 原生 async 支持，与现有异步 Agent 系统无缝集成 |
| 模板引擎 | Jinja2 | FastAPI 原生集成，服务端渲染 |
| CSS 框架 | Tailwind CSS (CDN) | 无需构建工具，快速实现设计系统 |
| 前端交互 | Vanilla JS + Fetch API | 轻量无依赖，聊天界面不需要重型框架 |
| 运行方式 | Uvicorn | FastAPI 标准 ASGI 服务器 |

### 字体

- **主字体**: DM Sans (Google Fonts), 后备 Inter / system-ui
- **字重**: 400 (正文), 500 (加粗), 600 (标题), 700 (强调)
- **行高**: 1.5 (正文), 1.3 (标题)

### 圆角

- 小元素 (徽章): 4-6px
- 卡片/面板: 12px
- 按钮: 8-10px
- 头像: 8-10px
- 消息气泡: 12px

### 动效

- 过渡: 150-200ms ease
- 悬停: 色彩/阴影变换，不缩放 (避免布局抖动)
- 加载: 脉冲动画 (pulse) + 打字机效果
- 消息出现: fade-in + slight slide-up

## 页面设计

### 1. 登录页 (/)

- **布局**: 居中卡片，垂直居中
- **内容**:
  - Hommey Logo + 品牌名称
  - "智能差旅助手" 副标题
  - 用户 ID 输入框 (聚焦态有蓝色光晕)
  - "进入系统" 按钮
  - 底部小字提示欢迎语
- **状态**: 空状态输入提示，错误提示 (ID 为空)
- **动效**: 卡片 fade-in，输入框 focus 动效

### 2. 主聊天页 (/chat/{user_id})

- **布局**: 三栏结构
  - 左侧栏 (64px): 导航图标
  - 中央栏 (flex-1): 聊天主区域
  - 右侧栏 (260px): 用户信息面板

#### 左侧导航栏
- Hommey 品牌图标 (顶部)
- 当前页面指示器 (蓝色竖条)
- 底部: 设置图标

#### 中央聊天区
- **顶部栏**: "Hommey 商旅助手" + 在线状态指示器 + 用户 ID
- **消息区**: 可滚动，flex 列，底部对齐
  - AI 消息: 左侧，蓝紫渐变头像，深色气泡，圆角 12px
  - 用户消息: 右侧，用户头像，蓝色气泡
  - Agent 状态条 (处理中): 显示调用的智能体标签 + 脉冲加载点
  - 打字机效果: 逐字显示 AI 回复
- **输入区**: 输入框 + 发送按钮
  - 输入框: 深色背景，圆角 10px，placeholder 文字
  - 发送按钮: 蓝色圆形，向上箭头图标
- **加载状态**: 初始加载时显示骨架屏

#### 右侧用户面板
- **问候区**: "您好，X先生" + 头像首字母 + 会员等级
- **个人资料**: key-value 列表 (常驻地、出行偏好、常住酒店)
- **快捷入口**: 2x2 网格 (规划行程、查天气、差旅标准、历史行程)

## API 设计

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | / | 登录页 |
| POST | /login | 提交用户 ID，重定向到聊天页 |
| GET | /chat/{user_id} | 聊天主页 |
| POST | /api/{user_id}/init | 初始化用户实例 (异步) |
| GET | /api/{user_id}/status | 检查用户实例状态 |
| GET | /api/{user_id}/preferences | 获取用户偏好 |
| POST | /api/{user_id}/chat | 发送消息，返回 AI 回复 |

## 用户流程

```
打开页面 → 输入用户ID → 点击进入
    ↓
初始化系统 (显示加载状态)
    ↓
进入主界面 (AI 发送欢迎消息)
    ↓
用户输入消息 → 发送
    ↓
显示 Agent 处理状态 (标签 + 脉冲)
    ↓
AI 回复 (打字机效果)
    ↓
继续对话...
```

## 交互细节

1. **初始化加载**: 页面显示 "正在为您初始化系统..." + 旋转加载图标
2. **消息发送**: 按钮变为禁用 + 旋转状态，防止重复提交
3. **Agent 处理中**: 在 AI 气泡中显示正在调用的智能体名称标签 + 脉冲动画点
4. **打字机效果**: 回复逐字显示，模拟 AI 实时生成
5. **消息列表自动滚动**: 新消息出现时自动滚动到底部
6. **输入框自动聚焦**: 每次回复完成后自动聚焦输入框
7. **右键面板更新**: 对话过程中偏好变更时自动刷新右侧面板

## 注意事项 (Anti-patterns)

1. ❌ 不使用 emoji 作为 UI 图标 (使用 SVG/Unicode 符号)
2. ❌ 悬停效果不改变元素尺寸 (避免布局抖动)
3. ❌ 消息输入时不禁用提交按钮 (应显示 loading 状态)
4. ❌ 页面加载时不显示空白 (应显示骨架屏或加载指示器)
