# V2.3 Runtime 工厂与测试边界整理

> 更新日期: 2026-06-05  
> 涉及分支: `structure-cleanup`  
> 基准版本: V2.2 项目结构与配置安全优化  
> 目标: 在不大搬家的前提下，降低入口层重复初始化和旧测试路径耦合。

---

## 1. 核心改动

### 1.1 新增共享 runtime 工厂

新增 `runtime.py`，集中创建核心运行对象:

- AgentScope 初始化
- LLM model
- `MemoryManager`
- `IntentionAgent`
- `LazyAgentRegistry`
- `OrchestrationAgent`
- circuit breaker

CLI、FastAPI WebUI、MCP server 不再各自复制这段初始化流程。

### 1.2 新增 `settings.py`

新增 `settings.py` 作为仓库内安全配置模块，运行时代码改为从 `settings` 读取:

- `LLM_CONFIG`
- `SYSTEM_CONFIG`
- `RAG_CONFIG`
- `SKILL_CONFIG`
- `RESILIENCE_CONFIG`
- `MEMORY_CONFIG`
- `MCP_CONFIG`

`config.py` 继续作为本地私有文件保留并忽略，不作为干净 clone 的运行依赖。

### 1.3 MCP server 复用 runtime

`hommey_mcp/hommey_mcp_server.py` 初始化时保存共享 runtime 里的:

- `_agent_registry`
- `_orchestrator`

工具函数不再每次调用时重复创建 `LazyAgentRegistry` 和 `OrchestrationAgent`。

---

## 2. 测试边界整理

### 2.1 移除旧 Agent 直连导入

以下测试不再导入已迁移的旧模块路径:

- `agents.event_collection_agent`
- `agents.rag_knowledge_agent`
- `agents.information_query_agent`
- `agents.itinerary_planning_agent`

对应测试改为验证当前 Skill 插件结构和 registry 映射。

### 2.2 新增 Skill registry 测试

新增 `tests/test_skill_registry.py`，覆盖:

- `SkillLoader` 可发现 8 个运行时 Skill
- `LazyAgentRegistry` 暴露 Skill 名称和 legacy intent 名称

### 2.3 LLM 集成测试默认跳过

`test_intention_agent.py` 和 `test_memory_system.py` 增加开关:

```bash
HOMMEY_RUN_LLM_TESTS=true
```

未设置时默认跳过，避免普通本地测试依赖真实 API Key。

## 3. 结构影响

当前仍保持模块化单体目录，不做 `src/hommey/` 大迁移:

```text

runtime.py
settings.py
```

这次只收敛入口初始化和测试边界，避免过早引入复杂包结构。
