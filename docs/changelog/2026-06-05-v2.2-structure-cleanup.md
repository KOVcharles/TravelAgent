# V2.2 项目结构与配置安全优化

> 优化日期: 2026-06-05  
> 涉及分支: `structure-cleanup`  
> 基准版本: V2.1 FastAPI WebUI + 性能优化  
> 优化目标: 收敛项目结构风险、移除配置泄露隐患、提升本地开发与后续重构的工程稳定性

---

## 1. 背景问题

本轮检查发现项目不是单纯“目录多”，而是存在几个会影响长期维护和安全性的结构问题:

1. `config.py` 被 Git 跟踪过，且历史提交里出现过真实 API Key。
2. 项目自有目录 `mcp/` 与第三方协议包 `mcp` 同名，会遮蔽依赖包，导致 `agentscope` 导入 `mcp.types` 失败。
3. Skill 插件加载路径硬编码为 `.claude/skills`，后续迁移成本高。
4. `.agents/`、运行态记忆文件、大模型文件、测试结果等本地/生成内容容易被误提交。
5. Redis/PostgreSQL 依赖在模块顶层导入，即使使用默认本地后端，也可能因为缺少可选依赖导致导入失败。
6. `requirements.txt` 缺少 FastAPI WebUI、测试、YAML 解析等实际依赖。

## 2. 配置与密钥安全

### 2.1 改动

- 新增 `.env.example`，作为环境变量模板。
- 新增本地 `.env`，用于保存真实 API Key；该文件已被 `.gitignore` 忽略。
- 更新 `.gitignore`，忽略:
  - `.env`
  - `.env.*`
  - `config.py`
  - `config.local.py`
- 将 `config.example.py` 改为安全模板，不包含真实密钥，只从环境变量读取。

### 2.2 当前约定

仓库只提交:

```text
config.example.py
.env.example
```

本地保留但不提交:

```text
config.py
.env
```

### 2.3 安全影响

最新代码不再需要把真实 Key 写入仓库文件。  
历史提交中出现过的旧 Key 仍应在平台侧废弃/轮换，因为 Git 历史可能已经被 GitHub 或其他 clone 保留。

## 3. MCP 包名冲突修复

### 3.1 问题

原项目目录:

```text
mcp/
```

会遮蔽第三方 Python 包 `mcp`。当 `agentscope` 内部执行:

```python
import mcp.types
```

Python 可能优先加载本项目的 `mcp/` 目录，导致:

```text
ModuleNotFoundError: No module named 'mcp.types'
```

### 3.2 改动

将项目自有 MCP 集成目录改名为:

```text
mcp/ -> hommey_mcp/
```

并更新引用:

```python
from hommey_mcp.mcp_manager import MCPManager
from hommey_mcp.mcp_config import MCPConfig
from hommey_mcp.hommey_mcp_server import server
```

第三方协议包仍然使用:

```python
from mcp.server.fastmcp import FastMCP
```

### 3.3 影响

项目包和第三方依赖包的命名边界更加清晰，避免运行时导入冲突。

## 4. Skill 插件加载路径收敛

### 4.1 问题

原先 `LazyAgentRegistry` 和 `SkillLoader` 都硬编码 `.claude/skills`。这导致:

- 插件目录迁移成本高。
- `.agents/skills` 与 `.claude/skills` 并存时，容易误改错目录。
- 意图识别读取的 Skill 元数据和运行时加载的 Agent 可能不一致。

### 4.2 改动

新增配置:

```python
SKILL_CONFIG = {
    "root": os.getenv("HOMMEY_SKILLS_ROOT", ".claude/skills"),
}
```

`LazyAgentRegistry` 和 `SkillLoader` 统一从该配置读取 Skill 根目录。

默认仍保持:

```text
.claude/skills
```

### 4.3 影响

当前运行方式不变，但后续如果迁移到 `src/hommey/skills`，只需要改配置，不需要全项目替换硬编码路径。

## 5. 可选依赖延迟导入

### 5.1 Redis

原先 `context/short_term_memory.py` 顶层导入:

```python
import redis
```

即使默认使用 `memory` 后端，也会因为未安装 Redis 客户端导致导入失败。

现在改为仅在:

```python
backend == "redis"
```

时才导入 `redis`。

### 5.2 PostgreSQL

原先 `context/long_term_memory.py` 顶层导入:

```python
import psycopg
```

即使默认使用 `file` 后端，也会因为未安装 PostgreSQL 驱动导致导入失败。

现在改为仅在初始化 `PostgresLongTermMemory` 时导入 `psycopg`。

### 5.3 影响

默认本地开发路径更轻量。只要使用:

```env
HOMMEY_SHORT_TERM_BACKEND=memory
HOMMEY_LONG_TERM_BACKEND=file
```

就不需要 Redis/PostgreSQL 服务或驱动即可导入核心模块。

## 6. 忽略规则整理

新增忽略项:

```gitignore
.agents/
.env
.env.*
!.env.example
config.py
config.local.py
data/memory/*.json
data/memory_test/
data/models/
tests/results/
benchmark-results/
```

### 6.1 目的

- `.agents/`: 本地/生成的 Skill 副本，不作为当前运行时唯一来源。
- `.env`: 本地真实密钥文件。
- `config.py`: 本地配置文件。
- `data/memory/*.json`: 运行态用户记忆。
- `data/models/`: 大模型本地资产。
- `tests/results/`: 测试报告产物。
