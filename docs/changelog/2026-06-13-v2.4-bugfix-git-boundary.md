# v2.4 Bugfix and Git Boundary Cleanup

Date: 2026-06-13

## Changes

- Prevented failed itinerary planning responses from being saved into long-term trip history.
- Fixed MCP `plan_trip` itinerary extraction to match the current orchestration result shape.
- Added `httpx` to runtime dependencies because the weather query skill imports it directly.
- Added `data/memory/.gitkeep` so the memory directory can exist without tracking local JSON memory files.
- Added a separated WebUI onboarding module for first-run preference initialization.
- Added onboarding API endpoints so initial preferences are saved directly instead of going through chat/LLM parsing.
- Updated the WebUI onboarding flow to save explicit preference keys and values.
- Made empty WebUI preference summaries return a stable shape for new users.
- Added CLI first-run onboarding using the same separated preference initialization module.
- Moved onboarding into a project-level shared module instead of keeping it under WebUI.
- Improved CLI onboarding with selectable defaults, manual input fallback, and optional IP-based city detection.
- Added a fast rule-based intent router for common high-confidence requests.
- Added CLI long-term summary caching to avoid summarizing memory on every turn.
- Added lightweight timing logs for CLI, WebUI, and per-agent orchestration.
- Shortened the effective `IntentionAgent` prompt before LLM calls.
- Moved fast intent routing and onboarding helpers under `core/` as shared non-agent application helpers.
- Added per-agent duration fields to orchestration results and CLI agent-call display.
- Exposed per-agent duration metadata in WebUI manager responses for future UI display.
- Added a minimal engineered RAG package with schemas, document loading, chunking, Milvus storage, retrieval, and ingestion entrypoints.
- Rewired the ask-question RAG agent to use the shared `rag.KnowledgeRetriever` while keeping its external methods stable.
- Simplified the ask-question knowledge-base initialization script into a thin wrapper around `rag.ingestion`.
- Fixed RAG dependency declaration by installing `milvus-lite` explicitly instead of relying on an unsupported `pymilvus` extra.
- Ignored `data/rag_scratch/` for temporary local RAG experiments.
- Moved RAG source documents out of the ask-question Skill and into project-level `data/documents/`.
- Updated RAG defaults so source documents are read from `data/documents/` and Milvus data is stored under `data/rag_knowledge/`.
- Removed duplicated Milvus, embedding, BM25, and fusion-ranking implementation from the ask-question Agent.
- Reduced the ask-question Agent to a thin AgentScope adapter that extracts the query, calls `KnowledgeRetriever`, and formats the LLM prompt/response.
- Added in-process embedding model caching for RAG stores to avoid repeated model construction when retrievers are recreated.
- Made the RAG Agent report an empty knowledge base explicitly instead of saying no relevant knowledge was found.
- Fixed RAG Agent handling for streaming LLM responses so async generators are consumed into answer text instead of displayed as object strings.
- Loaded local raw HuggingFace embedding directories through an explicit `Transformer + Pooling` wrapper to avoid noisy SentenceTransformer fallback warnings.
- Disabled LLM streaming only during RAG answer generation to avoid provider stream-shape parsing errors such as missing `text` fields.
- Made RAG LLM generation failures fall back to retrieved knowledge snippets instead of returning an internal error to the user.
- Made RAG LLM response parsing ignore unsafe `__aiter__` checks on AgentScope response objects that raise `KeyError`.
- Added lightweight meal-expense query expansion and reranking so "餐补" routes toward "餐费/餐饮/午餐/晚餐/报销" policy chunks.
- Made RAG chunking topic-aware so FAQ entries, numbered policy sections, and section headings are less likely to be merged into unrelated chunks.
- Added RAG relevance filtering and off-topic penalties after hybrid retrieval so generic travel-subsidy or unrelated international-travel snippets are less likely to outrank directly matching policy.
- Added Windows Milvus Lite rebuild handling for manifest-replacement failures and local collection cleanup during knowledge-base re-ingestion.
- Fixed cumulative streaming LLM snapshots in the RAG answer parser so repeated partial answers are collapsed instead of duplicated in the final response.
- Changed RAG LLM failure behavior to return a user-facing generation-failure message instead of dumping raw retrieved knowledge snippets.
- Added global RAG answer normalization for contradictory "no information, but..." responses when retrieved evidence contains related policy content.
- Moved cheap fast intent routing ahead of long-memory context construction in both CLI and WebUI for context-free single-agent intents.
- Kept the original `IntentionAgent` fast routing as a fallback so non-CLI/Web callers still get the same rule-based shortcut.
- Added additional policy-routing keywords such as `餐补`, `餐费`, `餐饮`, `饭补`, `补助`, `津贴`, `住宿费`, `交通费`, `差旅费`, and `发票`.
- Added a WebUI streaming chat endpoint at `/api/{user_id}/chat/stream` using newline-delimited JSON events.
- Updated the WebUI chat frontend to consume the streaming endpoint with `ReadableStream`, render answer chunks incrementally, and preserve normal error recovery.
- Reduced long-term memory summarization windows from 50 chat messages to 20 for CLI, WebUI, and the shared memory-manager default.
- Added broader ignore rules for local caches, local DB files, and nested memory JSON files while explicitly preserving `data/memory/.gitkeep`.
- Verified that `.env` and `config.py` remain ignored and that tracked files do not contain real API keys.

## Reason

- Failed plan-trip runs can return an empty itinerary placeholder for display, but that should not become a real trip history record.
- The MCP tool was reading the old nested `result.data` shape while the orchestrator now emits `data` directly.
- Local memory JSON files are runtime state and should stay outside commits.
- First-run preference collection should be deterministic and isolated from the normal chat pipeline.
- Policy/RAG questions should not pay the long-memory summarization cost before a high-confidence rule can route them directly.
- WebUI users need visible progress while the backend is still doing RAG retrieval and LLM summarization.
- Large local memory files should have a smaller summarization window to reduce first-turn latency.
- GitHub publishing requires a clean boundary between source files, local runtime state, generated vector stores, and real secrets.

## Impact

- Trip history is less likely to be polluted by failed planning attempts.
- MCP trip planning can return the extracted itinerary instead of falling back to the whole orchestration payload.
- Fresh installs have the dependency needed for weather queries.
- New WebUI users can complete initial preference setup without relying on LLM preference extraction.
- CLI users get the same initial preference setup after runtime initialization.
- CLI onboarding is less typing-heavy and no longer shows empty default parentheses.
- Common requests can skip heavy LLM intent prompts and route directly to the right skill.
- CLI responses avoid one repeated memory-summary LLM call on most turns.
- Timing logs make slow stages visible before further optimization.
- CLI users can now see which agent ran and roughly how long that agent took.
- Root-level project files stay less noisy because functional helpers live in `core/`.
- RAG document ingestion and retrieval now have clearer module boundaries instead of living entirely inside the Skill script.
- Future RAG tools can reuse the same ingestion pipeline without duplicating document loading and chunking code.
- Skill directories no longer own RAG knowledge assets; they only keep skill definitions and execution scripts.
- The RAG ranking implementation now has a single owner in `rag/milvus_store.py`, reducing confusion and dead code in the Skill agent.
- Empty or failed RAG ingestion is easier to diagnose from the chat response.
- RAG answers no longer leak internal async generator object representations to CLI users.
- RAG startup no longer prints the misleading "No sentence-transformers model found" warning for the local BGE model directory.
- RAG answers remain usable when the provider's streaming response format is incompatible with AgentScope's parser.
- Meal expense questions are less likely to be ranked behind generic travel-subsidy or international-travel snippets.
- FAQ and policy retrieval is more stable because one chunk now usually represents one question or policy topic instead of a mixed multi-topic block.
- Rebuilding the local Milvus Lite collection on Windows is less likely to fail because stale local collection artifacts are cleaned up before retrying.
- RAG responses are less likely to repeat the same sentence many times when the provider returns cumulative stream snapshots.
- RAG "not found" wording is less contradictory when retrieved documents contain related standards but not the exact user phrase.
- Policy, weather, and chitchat requests can skip long-memory summarization and LLM intent recognition when a high-confidence fast route is available.
- WebUI users see streamed response chunks instead of waiting for the entire `/chat` JSON response to finish before anything renders.
- Long-memory summarization is cheaper and less likely to dominate the first WebUI policy question for users with large memory files.
- Local runtime artifacts and secrets have a clearer Git boundary before publishing the version to GitHub.
