# 面试问题与答案

> 本文档用于集中记录面试中常见、重点或容易回答不完整的问题，并为每个问题整理一份简洁、准确、可直接用于面试表达的参考答案。

## 写入说明

新增内容时，请严格按照“**面试问题 → 答案**”的顺序书写，确保结构统一、便于阅读和检索。

答案应遵循以下要求：

- **精简明确**：只保留回答问题所必需的信息。
- **直击要害**：优先说明核心概念、关键原理或最终结论。
- **避免长篇大论**：一般控制在一个短段落或 3～5 个要点内。
- **便于口述**：使用自然、清楚的表达，避免堆砌术语。
- **必要时举例**：仅在例子能明显帮助理解时添加，并保持简短。
- **保证准确**：不确定的内容应先核实，不要写入模糊或可能误导的结论。

## AI 写入规则

当使用 AI 新增或整理内容时，请遵守以下指令：

> 根据本文档的固定格式写入面试问题和答案。答案必须精简、准确并直击要害，优先给出核心结论，避免背景铺陈、重复解释和长篇大论。除非问题确有需要，否则答案不超过 5 个要点。

### 面试问题

多 Agent 系统中，如何做好失败兜底，并防止 Agent 无休止调用工具？

### 答案

核心是“**分级失败 + 有界执行**”：

- 统一 Agent 返回协议，区分 `completed`、`partial_failure` 和 `failed`。
- 关键 Agent 失败立即终止后续依赖；辅助 Agent 失败则保留成功结果并降级返回。
- 只对当前失败 Agent 的瞬时错误做有限重试，不重跑整条链路。
- 设置请求级调用预算和超时，限制 Agent、LLM、RAG、搜索及 MCP 的调用次数。
- 失败步骤不写入记忆，并记录错误码、调用次数和耗时，便于追踪。
